# Hook Form

Lightweight JavaScript library для автоматического перехвата всех HTML-форм на странице и отправки данных на webhook с автоматическим сбором UTM-параметров, куки и служебной информации.

## Быстрый старт

### Подключение на странице

```html
<!-- Основной скрипт -->
<script src="https://cdn.jsdelivr.net/gh/USER/hook-form@v1.0.0/hook-form.js"></script>

<!-- Инициализация -->
<script>
  HookForm.init({
    webhook: 'https://hook.make.com/xxx'
  });
</script>
```

**Важно:** скрипт должен быть подключен после загрузки DOM или в конце `<body>`.

## API

### HookForm.init(options)

Инициализирует скрипт и подключает обработчики ко всем формам на странице.

**Параметры:**
- `webhook` (string, обязательный) — валидный HTTP(S) URL endpoint для отправки данных
- `timeout` (number, опциональный) — таймаут для fetch запроса в миллисекундах (по умолчанию 10000 = 10 сек)

**Возвращаемое значение:**
- `true` если инициализация успешна
- `false` если есть ошибки (неправильный webhook URL и т.д.)

**Пример:**
```javascript
// Базовая инициализация
const success = HookForm.init({
  webhook: 'https://hook.make.com/webhooks/xxx'
});

// С опциональным таймаутом
HookForm.init({
  webhook: 'https://hook.make.com/webhooks/xxx',
  timeout: 5000  // 5 секунд вместо 10
});

if (success) {
  console.log('HookForm инициализирован');
} else {
  console.log('Ошибка инициализации');
}
```

**Обработка ошибок:**

Если `webhook` не передан:
```
HookForm: webhook не передан в init()
```

Если `webhook` имеет невалидный формат:
```
HookForm: webhook должен быть валидным HTTP(S) URL: invalid-url
```

Если на странице нет форм:
```
HookForm: на странице не найдено ни одной <form>
```

## Структура данных

### Все данные отправляются как JSON в теле POST-запроса

Объект содержит следующие поля:

#### Поля из формы
Все значения из `input`, `select`, `textarea` элементов, которые имеют атрибут `name`:

```json
{
  "name": "Иван Петров",
  "email": "ivan@example.com",
  "phone": "+380501234567",
  "message": "Здравствуйте!"
}
```

#### Служебные поля

| Поле | Описание |
|------|---------|
| `form_name` | Имя формы: значение атрибута `name`, затем `id`, затем порядковый номер `form_1`, `form_2` и т.д. |
| `page_url` | Текущий URL без UTM-параметров и фрагмента (`#anchor`) |

#### UTM параметры

Автоматически читаются из URL и сохраняются в куки на 30 дней. При следующих посещениях страницы без UTM в URL, значения будут восстановлены из куки.

| Поле | Описание |
|------|---------|
| `utm_source` | Источник трафика (google, facebook, email и т.д.) |
| `utm_medium` | Тип трафика (cpc, social, email и т.д.) |
| `utm_campaign` | Название кампании |
| `utm_content` | Тип контента или вариант объявления |
| `utm_term` | Ключевое слово (для поиска) |

#### Click ID параметры

Используются для отслеживания кликов в рекламных системах:

| Поле | Описание |
|------|---------|
| `gclid` | Google Click ID (от Google Ads) |
| `fbclid` | Facebook Click ID (от Facebook Ads) |
| `ttclid` | TikTok Click ID (от TikTok Ads) |

#### Facebook параметры

Автоматически читаются из браузерных куки, установленных Facebook Pixel:

| Поле | Описание |
|------|---------|
| `fbp` | Facebook Pixel ID (из куки `_fbp`) |
| `fbc` | Facebook Conversion API ID (из куки `_fbc`) |

### Пример полного JSON

```json
{
  "name": "Иван Петров",
  "email": "ivan@example.com",
  "phone": "+380501234567",
  "message": "Спасибо за информацию",
  "form_name": "contact-form",
  "page_url": "https://example.com/contact",
  "utm_source": "google",
  "utm_medium": "cpc",
  "utm_campaign": "summer_sale_2026",
  "utm_content": "banner_v2",
  "utm_term": "купить товар",
  "gclid": "Cj0KCQjwraqnBhDsARIsAEQv-hZjKxYz...",
```

**Важно:** пустые значения передаются как пустая строка `""`, не опускаются.

## События

Скрипт диспатчит кастомные события на элементе формы в момент отправки данных:

### hookform:success
Диспатчится после успешной отправки сетевого запроса на webhook.

```javascript
document.querySelector('#myForm').addEventListener('hookform:success', function(e) {
  console.log('Данные отправлены на webhook!');
  // Показать сообщение об успехе
  // Скрыть форму
  // Перенаправить пользователя
  alert('Спасибо! Мы получили вашу заявку');
});
```

### hookform:error
Диспатчится при ошибке отправки на webhook. Событие содержит информацию об ошибке в свойстве `detail.error`.

**Типы ошибок:**
- `Timeout после Xms` — истёк таймаут отправки
- `Сетевая ошибка или CORS проблема` — проблема с подключением или сетью
- `Ошибка в JSON данных` — ошибка при формировании данных
- `Неизвестная ошибка` — другая ошибка

```javascript
document.querySelector('#myForm').addEventListener('hookform:error', function(e) {
  const errorType = e.detail?.error || 'Неизвестная ошибка';
  console.error('Ошибка отправки:', errorType);
  
  // Показать сообщение об ошибке пользователю
  if (errorType.includes('Timeout')) {
    alert('Запрос занял слишком много времени. Попробуйте ещё раз.');
  } else if (errorType.includes('Сетевая')) {
    alert('Проблема с интернетом. Проверьте подключение.');
  } else {
    alert('Ошибка при отправке данных. Попробуйте позже.');
  }
});
```

## Защита от ошибок

### Double Submit Protection
Скрипт автоматически защищает от повторной отправки одной формы. Если пользователь кликнет на кнопку отправки несколько раз подряд, данные будут отправлены только один раз.

Флаг `isSubmitting` остаётся установленным **до полного завершения** fetch запроса (успех или ошибка), даже если он идёт долго. Это гарантирует, что на медленном интернете пользователь не сможет отправить форму дважды.

При попытке двойного отправления в консоль выведется:
```
HookForm: Форма уже отправляется, дождитесь завершения
```

### Таймаут
По умолчанию скрипт ждёт ответа от webhook 10 секунд. Если за это время соединение не установлено, отправка считается ошибкой и диспатчится событие `hookform:error`.

Вы можете изменить таймаут при инициализации:
```javascript
HookForm.init({
  webhook: 'https://hook.make.com/webhooks/xxx',
  timeout: 5000  // 5 секунд вместо 10
});
```

### Обработка ошибок
Скрипт различает несколько типов ошибок:
- **Timeout** — соединение с webhook не установлено за отведённое время
- **Сетевая ошибка** — проблема с интернетом или браузером
- **JSON ошибка** — внутренняя ошибка при формировании данных

Все ошибки с описанием логируются в консоль браузера для отладки.

### Совместимость с режимом no-cors
Скрипт использует `mode: 'no-cors'` для максимальной совместимости. В этом режиме:
- Браузер не отправляет заголовок `Content-Type: application/json` (запрещено в no-cors)
- JSON данные отправляются в теле запроса как обычный текст
- Большинство вебхуков (Make, Zapier) распознают JSON автоматически
- Скрипт не получает статус ответа сервера (200, 500 и т.д.), но гарантирует отправку данных

### Загрузка DOM
Скрипт автоматически ждёт, пока страница полностью загрузится (событие `DOMContentLoaded`), прежде чем подключаться к формам. Это работает даже если вы подключили скрипт в `<head>` без атрибута `defer`.

## Отправка на webhook

- **Метод:** POST
- **Content-Type:** application/json
- **CORS mode:** no-cors (нет требований к CORS заголовкам на webhook)
- **Таймаут:** 10 сек по умолчанию (можно изменить)

Пример curl:
```bash
curl -X POST https://hook.make.com/webhooks/xxx \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "Иван",
    "phone": "+380501234567",
    "form_name": "form-hero",
    "page_url": "https://example.com/landing",
    "utm_source": "google",
    "utm_medium": "cpc",
    "utm_campaign": "summer_2026",
    "utm_content": "",
    "utm_term": "",
    "gclid": "",
    "fbclid": "",
    "ttclid": "",
    "fbp": "fb.1.1234567890.1234567890",
    "fbc": ""
  }'
```
