# Changelog

Все известные изменения в этом проекте будут задокументированы в этом файле.

## [1.0.0] - 2026-05-05

### Добавлено

- Автоматический перехват всех HTML-форм на странице
- Сбор данных из всех input, select, textarea элементов
- Автоматическое чтение и сохранение UTM параметров из URL
  - utm_source, utm_medium, utm_campaign, utm_content, utm_term
- Автоматическое чтение и сохранение Click ID параметров
  - gclid (Google Click ID)
  - fbclid (Facebook Click ID)
  - ttclid (TikTok Click ID)
- Автоматическое чтение Facebook Pixel куки
  - fbp (из куки _fbp)
  - fbc (из куки _fbc)
- Сохранение UTM параметров в куки на 30 дней
- Отправка данных формы на webhook методом POST с JSON
- Поддержка режима 'no-cors' для webhook

### Безопасность и надёжность

- ✅ **Валидация webhook URL** — перед инициализацией проверяется, что URL имеет формат HTTP(S)
- ✅ **Таймаут для fetch** — по умолчанию 10 сек (можно переопределить при init)
- ✅ **Double submit protection** — автоматическая защита от повторной отправки формы
- ✅ **Защита от повторной обработки форм** — каждая форма обрабатывается только один раз
- ✅ **Детальная обработка ошибок** — различение сетевых ошибок, timeout и других проблем
- ✅ **Информативное логирование** — все ошибки с описанием выводятся в консоль

### События

- Кастомные события успешной отправки (`hookform:success`) и ошибки (`hookform:error`)
- События содержат информацию об ошибке в свойстве `detail.error`

### Прочее

- IIFE обертка для изоляции глобального скоупа
- Поддержка последних 2 версий браузеров (Chrome, Firefox, Safari, Edge)
- Русскоязычные комментарии в коде
- Возвращаемое значение из init() для проверки успеха инициализации
- Опциональный параметр `timeout` для переопределения таймаута
